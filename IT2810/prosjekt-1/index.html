CSS Flexbox and CSS Grid

As I was starting this project I had an idea of what the page should look like. It quickly became apparent that I needed a two-dimensional layout. CSS Flexbox is for one-dimensional layout and CSS Grid is for two-dimensional layout, hence I concluded that CSS Grid would be optimal for dividing my sections and no need for CSS Flexbox.

In this project the grid has two columns and sections either takes up one column or two columns (also based on resizing). By dividing the sections in the grid it makes it easier to have the same size, distance and centering (or easily editable).

HTML Canvas and SVG

The animation and interactivity is mostly the same for both images (same idea atleast). When hovering over one of the images it starts to animate and the user can click to change the color of the angry emoji. When not hovering the image restores back to the default happy emoji.

The SVG image was designed using Figma. Figma is a great tool which lets you design easily, create groupings, add IDs etc. The most important part for this project is that Figma can export the image as SVG. The animation/ effects is made by accessing the IDs of the SVG in CSS and implementing CSS animations/effects. JavaScript is used to change the color of the emoji.

HTML5 Canvas combined with JavaScript is used to create the second image. I used the requestAnimationFrame to redraw the whole image to get the animation to work. Variables that changes the X- and Y-axis and a variable to keep track of frames to create the up/down and left/right animation. JavaScript is used to change the color and for eventlisteners. See section below for reasoning that I did not use jQuery.

jQuery

jQuery is a JS Library that makes it easier to manipulate HTML files among other things. It offers a variety of features for simplicity. As stated in the Piazza Forum (see sources) jQuery is on it way out and have not been the focus for this project, hence I have chosen the minimum requirement for jQuery in the documentation section.

Cross Browser Testing

Cross browser testing is important to check the compatibility of the webpage on different browsers. By doing so one can check the base functionality, the layout and the responsiveness works as intended for different browsers.

For cross browser testing I have focused on Chrome and Firefox. I have tested the webpage on both of these browsers to check my webpage compatibility. In addition I tested it on my computer and a friends computer to check if it works for different resolutions aswell. One issue I found a bit late is when you click fast on one of the images. The web browser will then mark up nearby text and in Safari it marks up the whole SVG section. Unfortunately I did not get to dwell to much on this issue.
